# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) documenting significant technical decisions made during the project.

## What is an ADR?

An Architecture Decision Record captures a decision, its context, alternatives considered, and the rationale. ADRs serve as a permanent record for current and future contributors.

## ADR Format

Each ADR follows the [MADR](https://adr.github.io/madr/) template:

- **Title** — Concise decision description
- **Status** — Proposed, Accepted, Deprecated, Superseded
- **Context** — Why the decision is needed
- **Decision** — What was decided
- **Consequences** — Trade-offs and impact
- **Alternatives** — Options that were considered
